# План улучшения авторизации

## Текущее состояние
- Страница логина использует Tailwind, но имеет простой дизайн
- Middleware защищает только /day и /api/financial-data маршруты
- Главная страница доступна неавторизованным пользователям
- Нет защиты от авторизованных пользователей на странице логина

## План улучшений

### 1. Обновление дизайна страницы логина
- **Цель**: Создать современный минималистичный интерфейс без анимаций
- **Изменения**:
  - Улучшить компоновку и пространство
  - Использовать современные цветовые схемы
  - Добавить лучшую типологию и иерархию
  - Улучшить визуальную обратную связь для форм
  - Сохранить минималистичный подход без анимаций

### 2. Обновление middleware для защиты маршрутов
- **Цель**: Защитить все маршруты, кроме /login
- **Изменения**:
  - Обновить matcher в middleware для защиты всех маршрутов
  - Добавить исключение для /login и статических файлов
  - Убедиться, что неавторизованные пользователи перенаправляются на /login

### 3. Защита страницы логина от авторизованных пользователей
- **Цель**: Авторизованные пользователи не должны видеть страницу логина
- **Изменения**:
  - Добавить проверку сессии на странице логина
  - Перенаправлять авторизованных пользователей на главную страницу

## Ожидаемый результат
- Современная, минималистичная страница логина без анимаций
- Полная защита всех маршрутов - неавторизованные пользователи видят только страницу логина
- Авторизованные пользователи не могут попасть на страницу логина
- Улучшенный пользовательский опыт и безопасность
